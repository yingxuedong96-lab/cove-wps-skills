# 通用文档排版

## 使用方式

用户提供规范文档，Skill 自动提取规则并应用到当前文档。

---

## 执行步骤

### Step 1 — 生成排版配置

根据用户提供的规范文档内容，生成排版配置 JSON。

**⚠️ 禁止自己写代码读取文档！必须通过以下方式获取规范内容：**
1. 如果用户直接提供了规范文字 → 直接使用，生成JSON配置
2. 如果用户指定了规范文档文件 → **调用 parse-spec.js 脚本读取**
3. 如果用户说"当前文档就是规范文档" → **调用 parse-spec.js 不传参数**

**⚠️ 特殊规则处理（图片、表格、公式、页码）**：
这些规则从 specText 自动识别，不需要在 paragraphRules 中添加：
- "图片居中" → 自动居中所有图片段落
- "表格等宽/与页面等宽" → 自动设置表格宽度
- "跨页重复表头" → 自动设置表格首行重复
- "公式居中" → 自动居中公式段落
- "公式编号右对齐" → 自动设置公式编号右对齐（制表位）
- "页码居中/左对齐/右对齐" → 设置页码位置
- "页码阿拉伯/罗马/中文数字" → 设置页码格式

**⚠️ 关键规则（必须严格遵守）**：

1. **specText 必须完整**：将用户规范原文**完整**放入，一字不漏，不能截断或改写
2. **类型必须完整覆盖**：规范提到多少种类型，配置就必须有多少种，绝不能遗漏
3. **属性必须完整提取**：规范提到的字体、字号、对齐、段前段后、行距、缩进等，全部都要提取，不能遗漏
4. **不要添加未提及的类型**：规范没说四级标题，就不要添加 heading4
5. **加缩规则**：规范说"加粗"才设置 `bold: true`，说"不加粗"才设置 `bold: false`，都没说则不要设置 bold 属性（保持原文档样式）
6. 直接输出 JSON，不要写代码，不要解释

**类型映射表**：

| 规范中的叫法 | 配置中的类型名 | 说明 |
|-------------|---------------|------|
| 主标题、文档标题、报告标题 | docTitle | 文档第一个段落（如"机载控制系统飞行安全体系报告"） |
| 章标题、一级标题、标题一 | zhangTitle | 章节标题（如"1 引言"、"第1章"） |
| 二级标题、标题二 | heading2 | 如"1.1 设计依据" |
| 三级标题、标题三 | heading3 | 如"1.1.1 具体内容" |
| 四级标题、标题四 | heading4 | 如"1.1.1.1 详细说明" |
| 五级标题、标题五 | heading5 | 如"1.1.1.1.1 补充说明" |
| 正文、正文格式 | body | 普通段落 |
| 图名、图片名、图标题、图号 | figureCaption |
| 表名、表格名、表标题、表号 | tableCaption |
| 参考文献 | ref |

**字号对照表**：

| 中文字号 | 磅值(fontSize) |
|---------|---------------|
| 初号 | 42 |
| 小初 | 36 |
| 一号 | 26 |
| 小一 | 24 |
| 二号 | 22 |
| 小二 | 18 |
| 三号 | 16 |
| 小三 | 15 |
| 四号 | 14 |
| 小四 | 12 |
| 五号 | 10.5 |
| 小五 | 9 |

**配置格式**：

```json
{
  "version": "1.0",
  "specText": "用户提供的规范文档原文（必须完整，一字不漏）",
  "fontDefaults": { "fontCN": "宋体", "fontEN": "Times New Roman" },
  "paragraphRules": {
    ...
  }
}
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| fontCN | string | 中文字体名称（如：宋体、黑体、楷体、仿宋） |
| fontEN | string | 西文字体名称（如：Times New Roman） |
| fontSize | number | 字号磅值（小四=12，四号=14，三号=16，五号=10.5，小五=9） |
| bold | boolean | 是否加粗 |
| alignment | number | 对齐方式：0=左对齐，1=居中，2=右对齐，3=两端对齐 |
| spaceBefore | number | 段前间距（磅） |
| spaceAfter | number | 段后间距（磅） |
| firstLineIndent | number | 首行缩进（磅，正值=首行缩进，负值=悬挂缩进，2字符≈24磅，-2字符≈-24磅） |
| lineSpacingRule | number | 行距：0=单倍，1=1.5倍，2=2倍，4=固定值 |

### Step 2 — 调用脚本应用排版

**⚠️ 必须调用 executeFile，不要自己写代码。**

```yaml
executeFile:
  filePath: "skills/generalformat/scripts/format-engine.js"
  params:
    config: <上一步生成的JSON>
```

返回值示例：

```json
{
  "success": true,
  "applied": 5656,
  "elapsedMs": 2882,
  "typeCounts": { "zhangTitle": 20, "heading2": 421, "body": 5207 },
  "total": 9388
}
```

## 段落类型识别

脚本自动识别以下段落类型，并自动设置大纲级别（用于导航窗格和目录生成）：

| 类型 | 匹配规则 | 大纲级别 |
|------|---------|---------|
| zhangTitle | 第X章、数字+空格+汉字、一、二、三、 | 1级 |
| heading2 | 1.1、1.2 等二级编号 | 2级 |
| heading3 | 1.1.1、1.1.2 等三级编号 | 3级 |
| heading4 | 1.1.1.1 等四级编号 | 4级 |
| tableCaption | 表1、表2-1 等 | - |
| figureCaption | 图1、图2-1 等 | - |
| body | 其他正文段落 | - |

**大纲级别作用**：
- 在WPS导航窗格中显示文档结构
- 支持自动生成目录
- 快速定位和跳转

## 图片与表格处理

脚本支持以下图片和表格设置（自动从 specText 识别）：

| 规范文本关键词 | 功能 |
|--------------|------|
| 图片居中、图片居中对齐 | 设置所有图片段落居中 |
| 表格等宽、与页面等宽 | 设置表格宽度等于页面可用宽度 |
| 跨页重复、重复表头 | 设置表格首行跨页时重复 |

**公式处理**：

| 规范文本关键词 | 功能 |
|--------------|------|
| 公式居中 | 公式段落居中对齐 |
| 公式编号、编号右对齐 | 公式编号右对齐（通过制表位实现） |

**公式编号格式**：支持 `(X-Y)`、`(X.Y-Z)`、`(A1)` 等常见格式，如 `(3-1)`、`(3.1-1)`、`(A1)`。

## 示例

**示例1：完整规则**

规范文档：
> 章标题用黑体三号居中加粗段前24磅段后12磅，二级标题用楷体四号左对齐加粗段前12磅段后6磅，三级标题用宋体小四号左对齐不加粗段前6磅段后3磅，正文用楷体四号两端对齐首行缩进2字符1.5倍行距，图名用黑体小五号居中段前6磅段后6磅，表名用黑体小五号居中段前6磅段后6磅。

生成的配置：

```json
{
  "version": "1.0",
  "specText": "章标题用黑体三号居中加粗段前24磅段后12磅，二级标题用楷体四号左对齐加粗段前12磅段后6磅，三级标题用宋体小四号左对齐不加粗段前6磅段后3磅，正文用楷体四号两端对齐首行缩进2字符1.5倍行距，图名用黑体小五号居中段前6磅段后6磅，表名用黑体小五号居中段前6磅段后6磅。",
  "fontDefaults": { "fontCN": "宋体", "fontEN": "Times New Roman" },
  "paragraphRules": {
    "zhangTitle": { "fontCN": "黑体", "fontSize": 16, "bold": true, "alignment": 1, "firstLineIndent": 0, "spaceBefore": 24, "spaceAfter": 12 },
    "heading2": { "fontCN": "楷体", "fontSize": 14, "bold": true, "alignment": 0, "firstLineIndent": 0, "spaceBefore": 12, "spaceAfter": 6 },
    "heading3": { "fontCN": "宋体", "fontSize": 12, "bold": false, "alignment": 0, "firstLineIndent": 0, "spaceBefore": 6, "spaceAfter": 3 },
    "body": { "fontCN": "楷体", "fontSize": 14, "alignment": 3, "firstLineIndent": 24, "lineSpacingRule": 1 },
    "figureCaption": { "fontCN": "黑体", "fontSize": 9, "alignment": 1, "spaceBefore": 6, "spaceAfter": 6 },
    "tableCaption": { "fontCN": "黑体", "fontSize": 9, "alignment": 1, "spaceBefore": 6, "spaceAfter": 6 }
  }
}
```

**示例2：图片与表格设置**

规范文档：
> 图名用宋体小四号居中段前6磅段后6磅，表名用宋体小四号居中段前6磅段后0磅，图片居中对齐，表格与页面等宽且跨页重复表头。

生成的配置：

```json
{
  "version": "1.0",
  "specText": "图名用宋体小四号居中段前6磅段后6磅，表名用宋体小四号居中段前6磅段后0磅，图片居中对齐，表格与页面等宽且跨页重复表头。",
  "fontDefaults": { "fontCN": "宋体", "fontEN": "Times New Roman" },
  "paragraphRules": {
    "figureCaption": { "fontCN": "宋体", "fontSize": 12, "alignment": 1, "spaceBefore": 6, "spaceAfter": 6 },
    "tableCaption": { "fontCN": "宋体", "fontSize": 12, "alignment": 1, "spaceBefore": 6, "spaceAfter": 0 }
  }
}
```

**注意**：图片居中、表格等宽、跨页重复表头、公式排版会从 specText 自动识别，无需额外配置。
