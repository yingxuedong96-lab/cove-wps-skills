# 增资协议生成器

## 核心约束（最高优先级）

**注意：**
- 不要有添加任何额外的语句或者说明
- 仅做原文的映射，不要在生成的条款和协议里面添加任何新的语句，否则会导致严重问题

---

## 模式选择

| 模式 | 输出内容 | 触发关键词 |
|------|----------|-----------|
| **简版** | 仅15条正文条款 | "简版"、"简版增资协议"、"只要条款"、"只要正文" |
| **正式版** | 完整格式（封面+前言+鉴于+正文+签署页+附件） | "正式版"、"正式增资协议"、"完整版"、"签署版"或未指定（默认） |

**判断规则：**
- 包含"简版"且不包含"正式版" → 简版
- 包含"正式版"或未指定 → 正式版

---

## 执行步骤

### Step 1 — 读取用户文档

```
executeFile({ filePath: "scripts/read-source.js", params: {} })
```

### Step 2 — 映射生成15条正文条款

**必须先完成此步骤，将用户投资条款转换为标准15条条款。**

参考 `references/mapping-example.md` 的映射示例。

**提取信息：**

| 字段 | 说明 | 关键词 |
|------|------|--------|
| companyName | 标的公司名称 | 公司、目标公司、标的公司 |
| founderName | 创始股东名称 | 创始股东、创始人、实际控制人 |
| investorA | 投资人A名称 | 投资人A、领投方、投资人 |
| signDate | 签署日期 | 签署日期、交割日 |
| mainBusiness | 投资款用途 | 投资款用于、投资款用途、资金用途、主要用于 |

**15条条款：**

| 条号 | 标题 | 内容来源 |
|------|------|----------|
| 第一条 | 定义 | 标准模板 |
| 第二条 | 本次交易安排 | 用户原文（投资额） |
| 第三条 | 增资款用途 | 用户原文（投资款用途） |
| 第四条 | 交割先决条件 | 用户原文（交割条件） |
| 第五条 | 增资款支付与交割 | 标准模板 |
| 第六条 | 公司治理安排 | 用户原文（董事会+重大事项） |
| 第七条 | 投资人特别权利 | 用户原文（反稀释/优先认购等） |
| 第八条 | 回购条款 | 用户原文（回购条款） |
| 第九条 | 清算优先权 | 用户原文（清算优先权） |
| 第十条 | 陈述与保证 | 标准模板 |
| 第十一条 | 进一步承诺 | 标准模板 |
| 第十二条 | 违约责任 | 标准模板 |
| 第十三条 | 保密 | 标准模板 |
| 第十四条 | 争议解决 | 标准模板 |
| 第十五条 | 其他 | 标准模板 |

### Step 3 — 生成文档

**简版：**
```
executeFile({
  filePath: "scripts/write-agreement.js",
  params: { "content": "<15条正文条款>" }
})
```

**正式版：**
```
executeFile({
  filePath: "scripts/write-agreement-formal.js",
  params: {
    "parties": { "companyName": "...", "founderName": "...", ... },
    "clauses": "<15条正文条款>"
  }
})
```

---

## 映射规则

1. **原文优先**：用户原文有的内容，按原文表述，不得修改
2. **标准模板**：用户原文没有的内容，使用 `references/agreement-template.md` 中的标准表述
3. **占位符**：无法提取的信息使用【】

---

## 文档格式

### 简版
- 章节标题：微软雅黑、12pt、加粗、左对齐
- 正文内容：微软雅黑、12pt、左对齐

### 正式版
- 封面：微软雅黑、四号(14pt)、居中
- 正文：微软雅黑、9pt、行距25pt
- 章节标题：微软雅黑、9pt、加粗、居中
