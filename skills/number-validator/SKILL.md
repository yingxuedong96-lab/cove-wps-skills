# 文档编号校对 Skill

> 专业的文档编号层级校对工具，自动识别编号样式混用，用修订模式智能修正。

---

## 适用场景

- 公文、政府文件、国企报告的编号规范化
- 学术论文、技术文档的编号层级检查
- 任何需要严格编号规则的正式文档校对

---

## 编号规范（两种，不得混用）

### 一、中文公文式

```
一、一级标题
（一）二级标题
1. 三级标题
（1）四级标题
① 五级标题
```

### 二、数字层次式

```
1. 一级标题
1.1 二级标题
1.1.1 三级标题
1.1.1.1 四级标题
```

## 执行流程

### 第一步：检测文档大小

**类型**：executeFile  
**脚本**：`scripts/init-schedule.js`  
**说明**：检测文档字数，决定使用直接模式还是调度器模式

**参数**：无  
**返回**：
- `path: 'direct'` - 直接模式（小文档或选区）
- `path: 'schedule'` - 调度器模式（大文档）

---

### 第二步：提取标题列表

**类型**：executeFile  
**脚本**：`scripts/extract-titles.js`  
**说明**：遍历所有段落，提取带编号的标题，返回完整标题列表

**参数**：
- `useSelection: false`（调度器模式）或 `true`（直接模式）
- `startPara: 1`（调度器模式用）
- `endPara: total`（调度器模式用）

**返回**：
```javascript
{
  titles: [
    { index: 1, text: "一、引言", level: 1, paraIndex: 5 },
    { index: 2, text: "（一）背景", level: 2, paraIndex: 8 },
    ...
  ]
}
```

---

### 第三步：分析编号问题

**类型**：llm  
**说明**：调用子Agent分析标题编号的样式类型、检测混用、跳级、重复等错误

**输入**：标题列表 JSON  
**输出**：修正建议列表（格式见 `skill.json` 中的 `subAgentSystemPrompt`）

---

### 第四步：应用修正（修订模式）

**类型**：executeFile  
**脚本**：`scripts/apply-corrections.js`  
**说明**：开启修订模式，根据修正建议逐一修正标题编号

**参数**：
- `corrections: [...]` - 修正建议列表

**返回**：
```javascript
{
  success: true,
  appliedCount: 5,
  skippedCount: 0
}
```

---

### 第五步：生成校对报告

**类型**：executeFile  
**脚本**：`scripts/apply-corrections.js`（通过 `mode: 'report'` 参数调用）  
**说明**：在文档末尾插入校对报告，总结修正情况

**参数**：
- `corrections: [...]`
- `mode: 'report'`

---

## 异常处理

**未检测到编号标题时：**

> 未在文档中找到编号标题，请确认文档包含需要校对的编号内容。

**检测到编号混用时：**

> 文档混用了中文公文式和数字层次式两种编号样式，请选择统一使用一种样式后重新校对。

**文档为空时：**

> 当前文档为空，无法执行编号校对。

**修正失败时：**

> 部分标题修正失败，请手动检查修订记录。失败原因：{错误详情}
