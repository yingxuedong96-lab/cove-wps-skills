## 检查规则

### L-001: 术语一致性

检查文档中同一概念是否使用了不同的术语表达。例如：
- "混凝土" 与 "砼"
- "钢筋" 与 "钢材"
- "坝体" 与 "大坝"
- "渗流" 与 "渗透"

### L-002: 模糊引用

检查是否存在模糊或不明确的引用，例如：
- "上述方法" 但前面没有明确说明是哪种方法
- "有关规定" 但没有指明具体文件
- "如表所示" 但没有指明是哪个表
- "如前所述" 但不清楚具体指前面哪部分内容

### L-003: 图表引用完整性

检查文档中引用的图表是否存在：
- "如图1所示" 但文档中没有图1
- "见表2-1" 但文档中没有表2-1
- 引用了不存在的公式编号

### L-004: 繁体字检测

检查文档中是否存在繁体字，应统一使用简体中文：
- "築" 应为 "筑"
- "壩" 应为 "坝"
- "測" 应为 "测"
- "計" 应为 "计"

## 输出格式

**只输出 JSON 数组，不要任何其他内容！**

```json
[
  {
    "rule": "L-004",
    "name": "繁体字检测",
    "location": "第31段",
    "original": "本報告中所有計算數據均經過嚴格校核，確保準確性。",
    "suggestion": "本报告中所有计算数据均经过严格校核，确保准确性。"
  }
]
```

**字段说明**：
- `rule`: 规则ID（L-001/L-002/L-003/L-004）
- `name`: 规则名称
- `location`: 段落位置，格式为"第X段"
- `original`: 原文内容（包含问题的完整文字）
- `suggestion`: 建议修改内容

**无问题时输出**：`[]`

## 注意事项

1. 语义检查需要结合上下文理解，请仔细分析文档内容
2. 对于专业术语，需要判断是否为同一概念的两种表述
3. 引用检查时，注意图表编号格式可能有变化
4. 繁体字检测时，注意区分专业术语（如某些人名、地名）
5. **必须输出纯JSON数组，不要有任何markdown标记或其他文字**
